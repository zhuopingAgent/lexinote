# V0 Summary

## Iteration Scope

V0 only covers the `Japanese AI Word Explainer` flow:

- user enters a Japanese word
- system returns a basic dictionary entry
- system returns an AI explanation for Chinese native speakers

Out of scope in V0:

- auth
- multi-user support
- favorites
- history
- exercises
- review
- voice
- cloud deployment

## Main Work Completed

### 1. Backend skeleton and lookup flow

Implemented the full backend flow for `POST /api/words/explain`:

- validate request body
- normalize input
- look up Japanese dictionary data from PostgreSQL
- generate structured AI explanation
- return a combined response for the UI

Main files:

- `app/api/words/explain/route.ts`
- `features/word-lookup/application/WordLookupService.ts`
- `features/japanese-dictionary/application/JapaneseDictionaryService.ts`
- `features/japanese-dictionary/infrastructure/JapaneseDictionaryRepository.ts`
- `features/ai-explanation/application/AIExplanationService.ts`
- `features/ai-explanation/infrastructure/LlmClient.ts`

### 2. Centralized PostgreSQL access

Implemented a small `pg`-based database layer with centralized SQL:

- shared pool creation
- retryable DB initialization
- centralized query helper
- single schema source of truth

Main files:

- `shared/db/pool.ts`
- `shared/db/init.ts`
- `shared/db/query.ts`
- `shared/db/sql/schema.sql`
- `shared/db/sql/dictionary.sql.ts`
- `shared/db/sql/seed.sql`

### 3. Structured response contracts

Defined shared DTOs for:

- word lookup request
- normalized dictionary entry
- structured AI explanation
- final combined response

Main file:

- `shared/types/api.ts`

### 4. Frontend UI

Implemented a single-page UI for the V0 flow:

- top header
- centered word input
- search button
- empty state
- loading state
- error state
- success state with dictionary entry and AI explanation

Main files:

- `app/page.tsx`
- `app/layout.tsx`
- `app/globals.css`

### 5. AI explanation contract hardening

Moved from free-text explanation rendering to structured explanation fields:

- `actualUsage`
- `commonScenarios`
- `nuanceDifferences`
- `commonMistakes`

This removed the earlier frontend regex parsing fragility.
